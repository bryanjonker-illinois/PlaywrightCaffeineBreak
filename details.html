Some details about Playwright

Local web server

This can access your own web server by updating the playwright.config.js. Note that you need to have an NPM command; you can create a stub if you are using LiveServer.

Existing website

You can point to an existing website and run the tests. Good for final verification and regression tests.

Uses both javascript and typescript

I'm using javascript because I'm more familiar with this. However, this is typescript-enabled. Most examples are in typescript.

Can generate screenshots

This is useful to get visual tests. Recommend only using this as needed and as small of an area as possible, because this takes a while to run

Can run accessibility tests

Integrates with aXe toolkit to give you all the accessibility options

Can interact with the page

You can use this to test how your website runs (uses Playwright Locators). Can choose by the following:

  • Role
  • Text
  • Title
  • Test ID
  • CSS or XPath selector

All of these have their plusses and minuses. If possible, I would focus on the Test ID methodology.

Has "pre" and "post" steps that can be run on suites

You can use this if your tests require you to log into a page or run steps for setup

Can be run using NPM or NPX

The NPM option is important because you can trigger this automatically using a Github CI/CD. This can be run on demand (very useful if you are doing this on an existing server) or on code checkin.

Note that if you are doing this, you need to install this on a temp server or use another tool like vite.
